A compact, portable CHIP-8 interpreter and emulator implemented in modern C++.
This project provides a small, standards-conforming CHIP-8 virtual machine with
an SDL2-based frontend for display and keyboard input. It is intended as both
an educational reference and a usable emulator for classic CHIP-8 ROMs.

**Repository layout**

- `source/` — emulator and application sources (C++).
- `ROMs/` — example ROMs and demos you can run.
- `build/` — out-of-source build artifacts (generated by CMake).
- `test/` — test CMake targets and test sources.

**Status:** actively maintained, C++20, requires SDL2 for runtime.

## Features

- Full CHIP-8 interpreter core (memory, registers, stack, timers, display).
- SDL2-based windowing and keyboard input.
- Example/demo ROMs included in `ROMs/`.
- Portable build via CMake (Linux/macOS/Windows with appropriate toolchain).

## Requirements

- CMake >= 3.14
- A C++20-capable compiler (GCC, Clang, MSVC)
- SDL2 development libraries (headers + linker)

On Debian/Ubuntu you can install required packages with:

```bash
sudo apt update
sudo apt install build-essential cmake libsdl2-dev
```

## Building

Recommended out-of-source build (from project root):

```bash
mkdir -p build
cmake -S . -B build
cmake --build build --target CHIP8_EMULATOR -- -j
```

This produces an executable named `CHIP8_EMULATOR` (the CMake target is
`CHIP8_EMULATOR_exe`). The binary is placed in the build output directory.

## Running

Usage:

```bash
./CHIP8_EMULATOR <ROM file path>
```

Example (from repo root):

```bash
./build/CHIP8_EMULATOR ROMs/demos/Zero\ Demo\ \[zeroZshadow,\ 2007\].ch8
```

If you use the provided `ROMs/` folder, point the emulator at any `.ch8` file.

## Controls (keyboard mapping)

The CHIP-8 hexadecimal keypad is mapped to a standard keyboard as follows:

Keypad layout -> Keyboard

```
Keypad:              Keyboard:
1 2 3 C              1 2 3 4
4 5 6 D              Q W E R
7 8 9 E      =>      A S D F
A 0 B F              Z X C V
```

So, for example, CHIP-8 key `5` maps to `W` and CHIP-8 key `A` maps to `Z`.

## ROMs and Demos

This repository includes a collection of classic CHIP-8 ROMs in the `ROMs/`
directory. Use them to test rendering, input, and timing. If you add your own
ROMs, place them anywhere accessible and pass the path to the emulator.

## Development

- The core emulator code lives under `source/` and targets C++20.
- The project uses SDL2 for display/input and CMake for build configuration.

To run the project in developer mode with extra checks (if desired): see
`cmake/dev-mode.cmake` and enable `CHIP8_EMULATOR_DEVELOPER_MODE` when
invoking CMake.

## Testing

There is a `test/` CMake target and a simple test harness; run tests via CMake
or your chosen test runner if configured.

## Troubleshooting

- Build failures regarding SDL2 usually indicate missing development packages
	(install `libsdl2-dev` or equivalent).
- If the emulator exits with "Usage" message, ensure you pass a valid ROM file
	path on the command line.
